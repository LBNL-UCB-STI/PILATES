# Converting `run_info.json` to a DuckDB Database

This document outlines the steps to transform a `run_info.json` file, generated by a PILATES simulation run, into a queryable DuckDB database. This process involves three main steps: generating SQL schemas, initializing the database with these schemas, and selectively uploading data.

## Prerequisites

Before you begin, ensure you have:

*   A `run_info.json` file from a completed PILATES simulation.
*   The PILATES conda environment activated (or all necessary Python packages installed).
*   The `pilates` repository cloned and your current working directory is the root of the repository.

## Steps

Follow these steps in order to create and populate your DuckDB database:

### Step 1: Generate SQL Schemas from `run_info.json`

The `schema_generator.py` script analyzes your `run_info.json` file and creates placeholder SQL schema files for any new tables it discovers. These generated schemas are placed in the `pilates/database/schema/generated/` directory.

**Command:**

```bash
python pilates/database/schema_generator.py <path_to_your_run_info.json>
```

**Example:**

```bash
python pilates/database/schema_generator.py run_info_urbansim.json
```

This command will output messages indicating which schema files were generated or if existing ones were skipped.

### Step 2: Initialize the DuckDB Database Schema

The `init_db_schema.py` script creates a new DuckDB database (if it doesn't exist) and initializes its schema by executing all SQL files found in the `pilates/database/schema/` directory (including those generated in Step 1).

**Command:**

```bash
python pilates/database/init_db_schema.py <path_to_your_database.duckdb>
```

**Example:**

```bash
python pilates/database/init_db_schema.py my_simulation.duckdb
```

This will create `my_simulation.duckdb` (or connect to it if it already exists) and apply all necessary table definitions.

### Step 3: Selectively Upload Data to the Database

Finally, the `selective_uploader.py` script reads data from your `run_info.json` and associated data files, then uploads it into the initialized DuckDB database. You can specify which tables to upload. If no tables are specified, it will attempt to upload all approved tables found in the `run_info.json` that have corresponding schemas.

**Command:**

```bash
python pilates/database/selective_uploader.py <path_to_your_run_info.json> <path_to_your_database.duckdb> [--table <table_name>]
```

*   Replace `<path_to_your_run_info.json>` with the path to your `run_info.json` file.
*   Replace `<path_to_your_database.duckdb>` with the desired path for your DuckDB database file.
*   Use `--table <table_name>` multiple times to specify individual tables, or omit it to upload all approved tables.

**Examples:**

*   **Upload all approved tables:**

    ```bash
    python pilates/database/selective_uploader.py run_info_urbansim.json my_simulation.duckdb
    ```

*   **Upload only specific tables (e.g., `households` and `persons`):**

    ```bash
    python pilates/database/selective_uploader.py run_info_urbansim.json my_simulation.duckdb --table households --table persons
    ```

This process will populate your DuckDB database with the simulation data, making it ready for analysis.
